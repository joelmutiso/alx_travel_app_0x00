✈️ ALX Travel Booking API (Project alx_travel_app_0x00)This project implements the core relational database models and serializers for a travel booking platform using Django and Django REST Framework. It follows best practices for data modeling, serialization, and includes a custom management command to seed the database with sample data.⚙️ Setup and ConfigurationPrerequisitesPython (3.8+)Virtual Environment setup (assumed to be in venv/)InstallationActivate Environment:source venv/bin/activate
# or on Windows PowerShell: .\venv\Scripts\activate
Install Dependencies:# Includes Django, DRF, and the seeding tool
pip install django djangorestframework django-seed
Security Setup (.env):Ensure your project root (alx_travel_app_0x00/) contains a .env file with your SECRET_KEY and DEBUG variables defined.💾 Database SchemaThe application revolves around three core models defined in listings/models.py.1. python manage.py migrate
Run Seeder Command: Execute the custom management command.python manage.py seed
Seed Operation SummaryThe command performs the following steps:Users: Creates 5 sample User accounts if none exist.Listings: Creates 10 sample Listing entries, randomly assigning a host.Bookings: Creates 15 sample Booking entries, linking them to random users and listings.Reviews: Creates 20 sample Review entries, linking them to random users and listings, with ratings between 1 and 5.✨ API Data RepresentationThe serializers are configured in listings/serializers.py to prepare the data for API consumption:SerializerPurposeNested Data IncludedCustom FieldsListingSerializerPrimary Listing Detailreviews (nested list)average_rating (calculated field)BookingSerializerReservation Detailguest (minimal details)listing_titleReviewSerializerReview Detailreviewer (minimal details)-UserDetailSerializerHelper--
